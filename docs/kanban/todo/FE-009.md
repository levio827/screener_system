# [FE-009] Market Overview & Sector Analysis Page

## Metadata
- **Status**: TODO
- **Priority**: Medium
- **Assignee**: Frontend Team
- **Estimated Time**: 18 hours
- **Sprint**: Phase 2 Enhancement
- **Tags**: #frontend #market-analysis #visualization #sector
- **Dependencies**: BE-009 (Market Overview API)

## Description
Create a comprehensive market overview page that provides users with a bird's-eye view of the Korean stock market, including market indices, sector performance, top movers, and market sentiment indicators. This page helps users understand the overall market context before making screening or investment decisions.

## Problem Statement
- No centralized view of overall market health
- Users cannot see sector-level performance at a glance
- Missing context for individual stock analysis
- No way to identify hot sectors or market trends
- Difficult to understand market breadth (advancing vs. declining)

## Proposed Solution
Build a comprehensive market overview page (`/market`) featuring:
1. Real-time market indices (KOSPI, KOSDAQ, KRX100)
2. Interactive sector performance heatmap
3. Market breadth indicators
4. Top gainers/losers across markets
5. Most active stocks by volume
6. Historical market trends (30-day, 90-day)

### Data Types
```typescript
interface MarketIndex {
  code: string                        // 'KOSPI', 'KOSDAQ', 'KRX100'
  name: string
  current: number
  change: number
  change_percent: number
  high: number
  low: number
  volume: number
  timestamp: string
  sparkline: number[]                 // Last 30 data points
}

interface MarketBreadth {
  advancing: number
  declining: number
  unchanged: number
  ad_ratio: number
  sentiment: 'bullish' | 'neutral' | 'bearish'
  timestamp: string
}

interface SectorPerformance {
  code: string                        // 'tech', 'finance', etc.
  name: string
  change_percent: number
  market_cap: number
  stock_count: number
}

interface MarketMover {
  code: string
  name: string
  market: 'KOSPI' | 'KOSDAQ'
  current_price: number
  change_percent: number
  volume: number
}
```
